Understanding This Dashboard

This document explains the metrics, decisions, and architecture behind AgentCloud Analytics - a platform-wide observability layer for AI agent usage across your organization.

Problem Statement

Engineering and data teams are using AI agents at scale, but there is no single view into how agents perform, what they cost, and where quality is degrading. Without visibility, teams cannot optimize spend, cannot catch reliability regressions before users are impacted, and cannot make data-driven decisions about which agents are delivering ROI.

AgentCloud Analytics solves this by aggregating telemetry from all agent runs - token usage, latency, errors, human quality ratings - and surfacing actionable KPIs in a single dashboard accessible to engineering leads, finance, and executive stakeholders.

Sample KPI Definitions
KPI-10 Cost per Quality Point Derived
Formula
total_cost / (avg_quality_score * rated_run_count)
Current Value
$0.42
Why it matters
Combines cost efficiency and output quality into a single optimization target. Lower values mean the platform is delivering more value per dollar spent.
Alert Threshold
Amber if > $0.60 for 3 consecutive days. Red if > $0.80.
Data Source
run_events + quality_ratings tables, 30-day rolling window

Platform Engineering
KPI-32 Platform Availability Reliability
Formula
(total_minutes - incident_minutes) / total_minutes * 100

Tracks the proportion of time the agent execution infrastructure is fully operational. Correlated with team productivity and user trust.

Architecture Decisions
D-2 Chart Library = Visx (Airbnb) Final
Decision
Use @visx/xychart and related Visx primitives for all data visualizations instead of Recharts or Chart.js.
Decided
Jun 3, 2026 - Platform Team
Rationale
Visx exposes D3 primitives via React components without abstracting away the SVG. This allows pixel-level customization for the heatmap calendar and stacked area charts while keeping bundle size minimal. Recharts hides too much internals; Chart.js is canvas-based which complicates SSR and accessibility.
Alternatives Considered
Recharts - rejected (limited customization)
Chart.js - rejected (canvas, accessibility)
Nivo - considered, heavier bundle
